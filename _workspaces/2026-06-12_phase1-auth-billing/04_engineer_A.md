## Key technical decisions (deviations from spec, all safer)
1. **D1 wiring simplified**: better-auth 1.6 `@better-auth/kysely-adapter` has a BUILT-IN D1 path — passing the raw `D1Database` binding as `database: env.DB` auto-selects its internal `D1SqliteDialect`. So I did NOT manually wire `kysely`+`kysely-d1`+`kyselyAdapter` as the spec sketched. Result: less code, no `CamelCasePlugin` anywhere (the exact risk in spec §13 / discussion #7487 is avoided by construction). `kysely`/`kysely-d1` still installed per spec for any repo-layer use by B/C.
2. **`@better-auth/cli` version**: spec said pin to better-auth's version, but the CLI has no stable 1.6.x — latest stable is 1.4.22. CLI runs as separate tool against an in-memory sqlite (`auth-cli.ts`); generated columns match 1.6 core. Acceptable.
3. **zod v4**: better-auth peer requires `zod ^4.3.6`, so the project resolves zod **v4** (not v3 as spec table said). Flagged to B/C in contract.
4. **Lazy router mounting**: `app.ts` loads B/C routers via dynamic import so A ships/type-checks before B/C exist; missing router → 501, no edit needed when they land.
5. **Header props**: A's `+layout.svelte` casts Header to a typed `Component<HeaderProps>` so it type-checks before C adds `$props()`. To remove once C wires Header.
